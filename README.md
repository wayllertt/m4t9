# Задание 9 — Параллельная загрузка погоды и финальный отчёт

Реализован шаблон Android-решения на `WorkManager`:

- Кнопка **«Собрать прогноз»** запускает параллельные `FetchCityWeatherWorker`.
- Каждый `Worker` переводится в foreground через `setForeground(...)` и держит ongoing-уведомление.
- `MainActivity` подписана на статусы unique work и обновляет текст/уведомление по прогрессу.
- `FinalizeReportWorker` запускается после всех загрузок, объединяет результаты и публикует финальное сообщение со средней температурой.

## Ключевые файлы

- `MainActivity.kt` — кнопка запуска и отображение статуса.
- `WeatherOrchestrator.kt` — сбор графа работ (`parallel -> final`).
- `FetchCityWeatherWorker.kt` — фоновая «загрузка» по городу.
- `FinalizeReportWorker.kt` — финальная агрегация.

## Важно

Это учебная заготовка: вместо реального API использована имитация задержки и случайные значения температуры/условий.
